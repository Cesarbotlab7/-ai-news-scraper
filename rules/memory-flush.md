# Memory Flush（常驻层）

> 不等用户触发——随时落盘。用户随时可能关窗。

## 触发条件

- **非 trivial 任务开始** → 立即在当天日志写会话头：`### HH:MM [项目] 正在做 XXX...`（崩溃恢复锚点，完成后填详情）
- 每个任务完成 → 更新当天日志对应条目，写结果和关键决策
- 每次 code commit → 在当天日志记录 commit 摘要
- 重要架构/策略决策 → 立即记入当天日志

## 退出信号（收到即执行完整 Flush）

触发词：「先这样」「今天就这样」「我先关了」「出门了」「明天再说」「关窗」「拜拜」「下线了」

收到退出信号后立即：
1. 更新当天日志：写当前进度、关键决策、待续事项
2. 确认 MEMORY.md 的 auto-memory 条目是否需要补充
3. 检查是否有未 commit 的改动，提醒用户

禁止：等用户手动触发才保存 / 攒批处理 / 假设用户会正常退出

## 日志文件规则

路径：`~/.claude/projects/-Users-lee/memory/YYYY-MM-DD.md`（用当天实际日期命名，如 `2026-04-30.md`）

- 每天一个文件，文件名即日期，不用 today.md
- 写入前先检查文件是否存在，存在则追加，不存在则新建

```
# YYYY-MM-DD

### HH:MM [项目名] 做了什么
- 结果：...
- 关键决策：...
- 待续：...

### HH:MM [项目名] ...
```
